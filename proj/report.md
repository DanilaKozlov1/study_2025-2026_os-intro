# Цель работы

Разместить заготовку персонального сайта научного работника на хостинге GitHub Pages с использованием генератора статических сайтов Hugo и шаблона Hugo Academic Theme.

# Задание

1. Установить необходимое программное обеспечение.
2. Скачать шаблон темы сайта.
3. Разместить его на хостинге git.
4. Установить параметр для URLs сайта.
5. Разместить заготовку сайта на Github pages.

# Теоретическое введение

Для реализации сайта используется генератор статических сайтов Hugo. В качестве шаблона индивидуального сайта используется шаблон Hugo Academic Theme. Репозиторий шаблона расположен по адресу: https://github.com/wowchemy/starter-hugo-academic.

GitHub Pages — сервис хостинга статических сайтов, который берёт файлы из репозитория на GitHub и публикует их в интернете. Сайт автоматически обновляется при каждом push в репозиторий благодаря GitHub Actions.

# Выполнение лабораторной работы

## 1. Установка Hugo

Так как используется операционная система Fedora Linux, установка Hugo производится через менеджер пакетов `dnf`:
```bash
sudo dnf install hugo
```

После установки была проверена версия:
```bash
hugo version
```

Вывод подтвердил наличие расширенной версии: `hugo v0.152.2+extended linux/amd64`.

## 2. Клонирование шаблона сайта

Был склонирован шаблон Hugo Academic и инициализированы подмодули:
```bash
git clone https://github.com/wowchemy/starter-hugo-academic.git my-site
cd my-site
git submodule update --init --recursive
```

## 3. Привязка к собственному репозиторию

На GitHub был создан репозиторий с именем `DanilaKozlov1.github.io`. Затем репозиторий шаблона был отвязан и заменён на собственный:
```bash
git remote remove origin
git remote add origin https://github.com/DanilaKozlov1/DanilaKozlov1.github.io.git
git push -u origin main
```

## 4. Настройка URL сайта

В файле `config/_default/hugo.yaml` был изменён параметр `baseURL`:
```yaml
baseURL: 'https://DanilaKozlov1.github.io/'
```

Изменения были зафиксированы и отправлены в репозиторий:
```bash
git add .
git commit -m "Set base URL"
git push
```

## 5. Проверка публикации на GitHub Pages

В настройках репозитория (Settings → Pages) было подтверждено, что сайт успешно развёрнут. Сайт доступен по адресу: https://danilakozlov1.github.io/

# Выводы

В ходе выполнения первого этапа индивидуального проекта было установлено необходимое программное обеспечение, склонирован шаблон сайта Hugo Academic, создан репозиторий на GitHub и настроена публикация через GitHub Pages. Заготовка персонального сайта успешно размещена и доступна в интернете.
