# Теоретическое введение

В системе по умолчанию открыто три специальных потока: stdin (стандартный поток ввода, дескриптор 0), stdout (стандартный поток вывода, дескриптор 1) и stderr (стандартный поток вывода ошибок, дескриптор 2). Потоки можно перенаправлять с помощью символов `>`, `>>`, `<`, `<<`.

Конвейер (pipe, символ `|`) служит для объединения команд в цепочки, в которых результат работы предыдущей команды передаётся последующей.

Команда `find` используется для поиска файлов по заданным критериям. Команда `grep` позволяет фильтровать текст, находя строки, содержащие указанную подстроку.

| Команда | Описание |
|---------|----------|
| `find`  | Поиск файлов по имени, типу, размеру и другим критериям |
| `grep`  | Фильтрация текста по шаблону |
| `df`    | Показывает размер смонтированных разделов диска |
| `du`    | Показывает размер файлов и каталогов |
| `ps`    | Информация о запущенных процессах |
| `kill`  | Завершение процесса по его PID |
| `jobs`  | Список задач, запущенных в фоновом режиме |

# Контрольные вопросы

**1. Какие потоки ввода-вывода вы знаете?**

В системе существует три стандартных потока ввода-вывода: stdin (стандартный поток ввода, файловый дескриптор 0) — по умолчанию клавиатура; stdout (стандартный поток вывода, файловый дескриптор 1) — по умолчанию консоль; stderr (стандартный поток вывода ошибок, файловый дескриптор 2) — по умолчанию консоль.

**2. Объясните разницу между операцией > и >>.**

Оператор `>` перенаправляет вывод в файл, при этом если файл уже существует, он перезаписывается. Оператор `>>` также перенаправляет вывод в файл, но открывает файл в режиме добавления — новые данные дописываются в конец файла, не затирая существующее содержимое.

**3. Что такое конвейер?**

Конвейер (pipe) — это механизм, позволяющий объединять команды в цепочки с помощью символа `|`. При этом стандартный вывод (stdout) одной команды передаётся на стандартный ввод (stdin) следующей команды. Например, `ls -la | sort` передаёт вывод команды ls команде sort для сортировки.

**4. Что такое процесс? Чем это понятие отличается от программы?**

Программа — это файл, содержащий набор инструкций (исполняемый код), хранящийся на диске. Процесс — это экземпляр программы, загруженный в оперативную память и выполняющийся в данный момент. Одна программа может породить несколько процессов, каждый из которых имеет собственное адресное пространство, состояние и идентификатор (PID).

**5. Что такое PID и GID?**

PID (Process ID) — уникальный идентификатор процесса, присваиваемый системой каждому запущенному процессу. GID (Group ID) — идентификатор группы пользователей. Каждый пользователь в системе принадлежит к одной или нескольким группам, и GID определяет принадлежность к основной группе.

**6. Что такое задачи и какая команда позволяет ими управлять?**

Задачи (jobs) — это программы, запущенные из терминала в фоновом режиме с помощью символа `&`. Управлять задачами позволяет команда `jobs`, которая выводит список текущих фоновых задач. Для завершения задачи используется `kill %номер_задачи`. Команда `fg` переводит задачу на передний план, а `bg` — возобновляет остановленную задачу в фоне.

**7. Найдите информацию об утилитах top и htop. Каковы их функции?**

Утилита `top` — это интерактивный монитор процессов, который в реальном времени отображает список запущенных процессов, отсортированных по потреблению ресурсов (CPU, память). Он показывает общую загрузку системы, количество процессов, использование памяти и swap. Утилита `htop` — это улучшенная версия top с более наглядным интерфейсом: цветное отображение, возможность прокрутки списка процессов, древовидное представление процессов, а также возможность отправки сигналов процессам прямо из интерфейса.

**8. Назовите и дайте характеристику команде поиска файлов. Приведите примеры использования этой команды.**

Команда `find` предназначена для поиска файлов и каталогов по различным критериям. Формат: `find путь [-опции]`. Основные опции: `-name` — поиск по имени, `-type` — по типу (f — файл, d — директория), `-size` — по размеру, `-user` — по владельцу, `-exec` — выполнение команды над найденными файлами. Примеры: `find ~ -name "*.txt"` — найти все .txt файлы в домашнем каталоге; `find /etc -name "h*" -print` — найти файлы в /etc, начинающиеся с h; `find ~ -type d` — найти все директории в домашнем каталоге.

**9. Можно ли по контексту (содержанию) найти файл? Если да, то как?**

Да, для поиска файлов по содержимому используется команда `grep`. Например, `grep -r "текст" ~/` рекурсивно ищет строку «текст» во всех файлах домашнего каталога. Опция `-l` позволяет вывести только имена файлов, содержащих указанную строку: `grep -rl "текст" ~/`.

**10. Как определить объём свободной памяти на жёстком диске?**

Для этого используется команда
`df`. Например, `df -h` выводит информацию о смонтированных разделах в удобочитаемом формате (в гигабайтах и мегабайтах), включая общий размер, занятое и свободное пространство.

**11. Как определить объём вашего домашнего каталога?**

С помощью команды `du`. Например, `du -sh ~/` выводит суммарный размер домашнего каталога в удобочитаемом формате. Опция `-s` означает суммарный итог, `-h` — человекочитаемый формат.

**12. Как удалить зависший процесс?**

Сначала нужно узнать PID процесса командой `ps aux | grep имя_процесса` или с помощью `top`/`htop`. Затем завершить процесс командой `kill PID`. Если процесс не реагирует на обычный сигнал, используется принудительное завершение: `kill -9 PID` (сигнал SIGKILL). Также можно использовать команду `killall имя_процесса` для завершения всех процессов с данным именем.
