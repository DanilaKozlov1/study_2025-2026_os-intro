# Теоретическое введение

Файловая система Linux — иерархическая структура каталогов и файлов. Основные команды для работы с файлами: `touch`, `cat`, `less`, `head`, `tail`, `cp`, `mv`, `chmod`. Права доступа задаются для трёх категорий: владелец (u), группа (g), остальные (o) — и включают чтение (r), запись (w), выполнение (x).

| Имя каталога | Описание каталога                                                                                                          |
|--------------|----------------------------------------------------------------------------------------------------------------------------|
| `/`          | Корневая директория, содержащая всю файловую систему                                                                       |
| `/bin`       | Основные системные утилиты                                                                                                 |
| `/etc`       | Общесистемные конфигурационные файлы                                                                                       |
| `/home`      | Домашние директории пользователей                                                                                          |
| `/tmp`       | Временные файлы                                                                                                            |
| `/usr`       | Вторичная иерархия для данных пользователя                                                                                 |

## 3. Установка прав доступа
```bash
mkdir australia && chmod 744 australia   # drwxr--r--
mkdir play      && chmod 711 play        # drwx--x--x
touch my_os     && chmod 544 my_os       # -r-xr--r--
touch feathers  && chmod 664 feathers    # -rw-rw-r--
```

## 4. Упражнения с правами

Просмотрен файл `/etc/passwd` командой `cat`. Файл `feathers` скопирован в `file.old`, перемещён в `~/play`. Каталог `~/play` скопирован в `~/fun`, затем перемещён в `~/play/games`.

При лишении владельца права на чтение (`chmod u-r ~/feathers`) команды `cat` и `cp` выдают ошибку `Permission denied`. После возврата права (`chmod u+r ~/feathers`) файл снова доступен.

При лишении владельца права на выполнение каталога `~/play` (`chmod u-x
~/play`) переход в каталог командой `cd` невозможен — выдаётся `Permission denied`. После возврата права (`chmod u+x ~/play`) переход снова работает.

## 5. Команды mount, fsck, mkfs, kill
```bash
man mount
man fsck
man mkfs
man kill
```

- `mount` — монтирует файловую систему устройства к дереву каталогов. Пример: `mount /dev/sdb1 /mnt`
- `fsck` — проверяет и восстанавливает целостность файловой системы. Пример: `fsck /dev/sda1`
- `mkfs` — создаёт файловую систему на устройстве. Пример: `mkfs.ext4 /dev/sdb1`
- `kill` — завершает процесс по PID. Пример: `kill 1234`, принудительно: `kill -9 1234`

# Ответы на контрольные вопросы

1. На жёстком диске используется файловая система ext4 — современная журналируемая файловая система Linux с поддержкой больших файлов и разделов.

2. Структура первого уровня: `/` — корень, `/bin` — утилиты, `/etc` — конфиги, `/home` — пользователи, `/tmp` — временные файлы, `/usr` — программы, `/var` — переменные данные, `/dev` — устройства, `/proc` — процессы.

3. Чтобы содержимое файловой системы стало доступно, её необходимо смонтировать командой `mount`.

4. Основные причины нарушения целостности: некорректное отключение питания, аппаратные сбои. Устраняется командой `fsck`.

5. Файловая система создаётся командой `mkfs`. Пример: `mkfs.ext4 /dev/sdb1`.

6. Команды просмотра текстовых файлов: `cat` — вывод целиком, `less` — постранично, `head` — первые строки, `tail` — последние строки.

7. `cp` — копирует файлы и каталоги. Опции: `-r` (рекурсивно), `-i` (с подтверждением), `-f` (принудительно).

8. `mv` — перемещает и переименовывает файлы и каталоги. Опции: `-i` (с подтверждением), `-f` (принудительно).

9. Права доступа — разрешения на чтение (r), запись (w) и выполнение (x) для владельца, группы и остальных. Изменяются командой `chmod`. Пример: `chmod 755 file` или `chmod u+x file`.
